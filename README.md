Assignment 2 – Banking System

Student: Darkhan Daurambekov | SE-2512

This project simulates a simple banking system using Java data structures.
It demonstrates the use of both logical and physical data structures.


Data Structures Used
1. LinkedList

Used to store all bank accounts.
Allows dynamic adding, searching, and updating accounts.

2. Stack

Used to store transaction history.
Implements LIFO (Last In First Out):

push → add transaction
pop → undo last transaction
peek → view last transaction
3. Queue

Used in two parts:

Bill payments → processed in FIFO order
Account requests → admin processes requests
4. Array

Used to store 3 predefined accounts (fixed size).
Demonstrates physical data structure.

Features
  Task 1 – Account Management
  Add new account
  Display all accounts
  Search account by name
Task 2 – Transactions
  Deposit money
  Withdraw money
  Balance update
Task 3 – Transaction History
  Store operations in Stack
  View history
  Undo last action
  View last transaction
Task 4 – Bill Payment Queue
  Add bill
  Process bill
  View queue
Task 5 – Account Request System
Submit account request
  Admin processes request
  Move account to main list
Task 6 – Array
  Store 3 predefined accounts
  Display them

Program Menu
  Bank Menu
    Submit account request
    Deposit / Withdraw
    Add bill
    View accounts
    View history
  ATM Menu
    Balance enquiry
    Withdraw
  Admin Menu
    View requests
    Process requests
    View bill queue
    Process bills

  How to Run
Open project in IntelliJ IDEA or VS Code
Compile and run Main.java
Use menu options to interact with the system

Conclusion

In this project, I learned how to:

Use LinkedList, Stack, Queue, and Array
Apply data structures to real-life scenarios
Understand logical vs physical data structures
Build a simple interactive Java application
